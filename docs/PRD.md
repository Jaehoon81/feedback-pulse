# PRD: feedback-pulse

## 목표
1인 크리에이터가 YouTube URL만 붙여넣으면 시청자 반응(강점·개선점·감성·주제·주목 댓글)을 1분 안에 종합 리포트로 받아본다.

## 사용자
- 자기 채널 영상의 시청자 피드백을 빠르게 정리하고 싶은 1인 크리에이터
- 댓글을 일일이 다 읽기엔 시간이 부족하지만 패턴은 알고 싶은 사람

## 핵심 기능 (10개, F-01 ~ F-10)

1. **F-01 URL 입력 받기** — 홈 폼에서 다양한 YouTube URL 포맷(`watch?v=`, `youtu.be/`, `shorts/`, embed) 허용
2. **F-02 videoId 추출** — URL 파싱으로 11자리 videoId 분리
3. **F-03 영상 메타데이터 + 댓글 수집** — YouTube Data API v3로 영상 정보 + 댓글 **최대 200개**(`pageToken` 페이지네이션 2회, ADR-004) — Vercel Hobby 60초 한도 적합
4. **F-04 Gemini 분석** — 수집한 댓글을 Gemini API에 한 번에 전달, `responseSchema`(JSON 스키마)로 감성 분포 + 강점/개선점/주제/주목 댓글을 동시에 추출
5. **F-05 6항목 리포트 렌더**:
   - 핵심 요약 (Executive Summary, 2~4문장)
   - 감성 분포 가로 막대 (긍정/중립/부정 비율, CSS — ADR-006 변경)
   - 주요 주제 (Topic Tags)
   - 강점 (잘하고 있는 점 + 인용 근거)
   - 개선점 (시청자 비판/요청 + 인용 근거)
   - 주목 댓글 (Notable Comments, 3~6개)
6. **F-06 localStorage 분석 히스토리** — 사이드바에서 과거 분석 다시 열람 (영구 보관)
7. **F-07 마크다운 다운로드** — 리포트를 `.md` 파일로 내보내기
8. **F-08 리포트 복사** — 핵심 요약 단독 복사 / 각 섹션(강점·개선점·주제·주목 댓글)별 마크다운 복사 (`navigator.clipboard.writeText` + `execCommand` fallback)
9. **F-09 메인 페이지 상태별 화면** — 5가지 상태(초기 / 입력 진행 / 분석 중 / 에러 / 완료)에 따른 정확한 레이아웃 정의
10. **F-10 리포트 페이지 진입 경험** — 스켈레톤 → 순차 fade-in으로 섹션 등장, 모바일(<768px)에서 6항목 Collapsible 접이식 (기본 펼침)

## 사용자 흐름
1. 홈에서 YouTube URL 입력 → "분석" 버튼
2. 진행 상태 표시(댓글 수집 → Gemini 분석) → 리포트 화면 전환
3. 좌측 사이드바에 과거 분석 목록, 클릭하면 다시 열람
4. 리포트 상단 "마크다운 다운로드" 버튼으로 `.md` 파일 저장

## MVP 제외 사항
- 로그인 / 계정 / 사용자 데이터 동기화
- 다중 영상 일괄 분석
- 채널 단위 트렌드 분석 (시간순 비교)
- 답글(reply) 수집
- 다국어 UI (한국어 단일)
- 댓글 원문 페이지네이션/검색 (분석된 인용만 보여줌)

## 디자인 방향
- 다크 모드와 라이트 모드 동등 지원
- 무채색 베이스 + 시맨틱 색상 (긍정 초록 / 부정 빨강 / 중립 회색)
- 도구처럼 보일 것 — 마케팅 페이지가 아니라 매일 쓰는 대시보드
- 데이터가 주인공, 장식 최소

## 성공 지표 (MVP)
- URL 붙여넣기 → 리포트 확인까지 60초 이내
- 4종 시나리오(정상 / 댓글 비활성 / 비공개 / 잘못된 URL) 모두 적절한 UI 응답
- 콘솔 에러 0건, 라이트/다크 + 데스크톱/모바일 시각적 정상

## 비기능 요구사항

- **성능**: URL 입력 → 리포트 완료까지 **P50 30초 / P90 50초 / P99 60초** 목표 (Vercel Hobby 60초 한도, ADR-026)
- **용량**: localStorage 히스토리 **최대 50건** (FIFO 자동 삭제)
- **브라우저 지원**: Chrome / Firefox / Safari 최신 2 메이저 버전, 모바일 Chrome/Safari 동일
- **접근성**: 키보드 네비게이션 가능, WCAG AA color contrast, 색상만으로 정보 전달 금지
- **오프라인**: 새 분석은 불가, 과거 리포트 열람·다운로드는 가능

## 제약 조건

- **배포 환경**: **Vercel Hobby (Free) 플랜** — Function `maxDuration` **60초**, 100k invocations/일, 100GB bandwidth/월, 6,000 build-min/월 (ADR-026)
- **YouTube Data API v3 일일 쿼터**: 10,000 units. 영상 1건당 `videos.list`(1 unit) + `commentThreads.list`(페이지당 1, 최대 2) ≈ **3 units** → 일 약 3,300 영상 분석 한도
- **Gemini API 비용**: 무료 티어 내 동작 — Gemini 2.5 Flash 250 RPD / 10 RPM (ADR-011 — 2026-05-29 pro→flash 변경, pro는 무료 티어 input token/day=0으로 사용 불가). 1인 사용량(하루 5~20회) 안에 충분. 한국어 분석 품질 부족 시 Claude Sonnet 4.6 fallback (호출당 ~$0.020, 유료)
- **localStorage 용량**: 브라우저 한도 ≈ 5MB. 리포트당 평균 30KB 가정 → 약 170건 한도. 50건 정책은 안전 마진

## 비용 추정 (개발자용)

기본 채택은 **Gemini 2.5 Flash 무료 티어** (ADR-011 — 2026-05-29 pro→flash 변경). 1차 LLM 비용은 $0 (무료 한도 250 RPD / 10 RPM 안에서). 한국어 비꼼/반어 인식 부족이 검증되면 Claude Sonnet 4.6 fallback. 아래 표는 비교용으로 두 모델 모두 표기.

기준: 한국어 댓글 평균 50자(≈100 tokens), 시스템 프롬프트 + 메타데이터 + `responseSchema` ≈ 3K 고정 토큰. Claude Sonnet 4.6 가격 가정($3/1M 입력, $15/1M 출력).

| 댓글 수 | 입력 토큰 (추정) | 출력 토큰 (추정) | Gemini 2.5 Flash 비용 | Claude Sonnet 4.6 비용 (fallback) | YouTube 쿼터 (units) |
|---|---|---|---|---|---|
| 50 | ~5K | ~1.5K | **$0** (무료) | ~$0.005 | 2 (`videos`+1페이지) |
| 100 | ~10K | ~1.8K | **$0** (무료) | ~$0.010 | 2 |
| **200 (MVP 상한)** | ~20K | ~2K | **$0** (무료) | ~$0.020 | 3 (1 + 2페이지) |

- 일일 한도: YouTube 10,000 units → 약 3,300 영상(200개 댓글 기준). Gemini 2.5 Flash 250 RPD → 250 분석/일.
- 댓글이 더 적은 영상은 비용/토큰도 낮아짐 (페이지네이션 조기 종료).
- 향후 Vercel Pro 업그레이드 시 댓글 500개까지 확장 가능 (ADR-004 트레이드오프).
- Gemini 무료 한도 초과 시 동일 API 키로 결제 등록(Pay-as-you-go)하거나 Claude로 마이그레이션.

## 수용 기준 (기능별)

| 기능 | Acceptance Criteria |
|------|---------------------|
| 1. URL 입력 | 빈 입력 / 공백만 / 2000자 초과 URL은 즉시 거부 + inline 메시지 |
| 2. videoId 추출 | 5종 포맷(`watch?v=`, `youtu.be/`, `shorts/`, `&v=`, embed) 지원. 채널 URL / 재생목록 URL은 `InvalidUrlError` |
| 3. 수집 | 댓글 200개 미만 영상도 정상 처리, 댓글 0개는 `CommentsDisabledError`로 통합 |

| 6. 히스토리 | 분석 완료 즉시 사이드바 갱신, 50건 초과 시 가장 오래된 항목부터 제거 |
| 7. 다운로드 | 파일명 `{video-title}_{YYYYMMDD}.md`, 모든 6항목 포함 |

## 로딩/대기 UX

- URL 제출 즉시 진행 단계 표시:
  1. "메타데이터 수집 중..." (예상 ~5초)
  2. "댓글 N/200개 수집 중..." (페이지네이션 진행률 갱신, 최대 2회)
  3. "분석 중..." (예상 ~25~35초)
- **남은 시간 추정**: 진행 단계 가중치 + elapsed time 기반으로 "약 {N}초 남음" 표시 (ARCH "진행률 가중치 + 남은 시간 추정" 알고리즘 참조)
  - 30초 미만: "약 {N}초 남음"
  - 30~60초: "약 1분 남음"
  - 60초 초과: "1분 이상 소요 중" (구체 숫자 숨김)
- 분석 단계는 단색 회전 spinner 사용 (UI_GUIDE 룰 준수)
- 60초 이상 걸리면 "댓글이 많아 시간이 더 걸리고 있습니다" 추가 안내
- 사용자가 분석 도중 새 URL 제출 시 `AbortController`로 진행 중 분석 즉시 중단
- **분석 취소 버튼**: 진행 단계 라벨 옆에 "분석 취소" 텍스트 버튼(neutral 색상) 노출
  - 클릭 시 즉시 `AbortController.abort()` 호출
  - 폼 상태를 `idle`로 복귀, 부분 결과는 localStorage에 저장하지 않음
  - 버튼은 `analyzing` 단계까지만 활성, `saving` 이후엔 비활성 (저장은 빠르고 abort 의미 없음)

## 데이터 보존 정책

- 분석 히스토리 최대 50건 (FIFO 자동 삭제)
- localStorage 키: `feedback-pulse:reports:v1:{id}`, `feedback-pulse:history:v1`
- 사용자가 사이드바에서 항목별 명시적 삭제 가능
- 마크다운 다운로드 후에도 localStorage 항목은 유지 (별도 백업)
- 스키마 변경 시 `:v1` 접미사로 마이그레이션 슬롯 확보 (ADR-009)

## 분석 한계

- **언어**: Gemini가 한국어 / 영어 / 일본어 등 다국어 댓글을 자연스럽게 처리. 분석 결과는 한국어로 산출
- **이모지**: Gemini가 이모지의 감성 함의를 인식 (예: 😍 = 강한 긍정, 🤔 = 회의/궁금)
- **스팸/봇 댓글**: 별도 필터링 없음. Gemini가 자체 판단으로 무시하거나 "주목 댓글"에서 제외
- **답글(reply)**: 수집하지 않음 (MVP 제외 사항)
- **민감 정보**: 댓글 작성자 표시명(`displayName`)은 원문 그대로 표시 (YouTube에서 이미 공개된 정보)

## 엣지 케이스

| 케이스 | 처리 방식 |
|--------|----------|
| 댓글 0개 | 분석 안 함, `CommentsDisabledError`로 통합 처리 |
| 댓글 10개 미만 | 정상 분석 진행, 일부 항목(주제 등)은 "데이터 부족" 표시 |
| 외국어 댓글만 있는 영상 | Gemini가 그대로 분석, 결과는 한국어로 |
| 매우 긴 댓글 (5000자+) | 그대로 Gemini에 전달 (토큰 한도 내에서 합산) |
| 라이브 스트리밍 / 멤버십 전용 영상 | YouTube API가 거부 → `VideoNotFoundError` |
| 분석 중 영상이 삭제됨 | YouTube 호출에서 실패 → `VideoNotFoundError` |
| 채널 URL / 재생목록 URL | `extractVideoId`가 거부 → `InvalidUrlError` |
| 분석 도중 사용자가 페이지 떠남 | `AbortController`로 진행 중 fetch 중단, localStorage 저장 안 됨 |
| 동일 영상 재분석 | 새 uuid로 별도 저장 (이전 리포트 덮어쓰기 안 함) |
| localStorage `QuotaExceededError`(브라우저) | FIFO로 가장 오래된 리포트부터 삭제 후 재시도 |
| 연령제한 영상 | YouTube API가 401/403 반환 → `VideoNotFoundError`로 일반화 (anonymous API 접근 거부) |
| 이모지 전용 댓글 (`🔥🔥🔥`) | 그대로 Gemini에 전달. 강한 감성 신호로 인식 |
| 도배/스팸 댓글 (동일 텍스트 반복) | Gemini가 자체 판단으로 무시. 강한 패턴이면 주목 댓글에 1개 대표로 |
| 너무 짧은 댓글 (`ㅋㅋ`, `?`, 1-3자) | Gemini가 의미 추출 어려우면 분석에서 제외 |
| HTML 엔티티 포함 (`&lt;`, `&amp;`) | `services/youtube`에서 디코딩 후 Gemini에 전달 |
| 삭제된/숨겨진 댓글 (`heldForReview`) | YouTube API가 반환하지 않음 → 자동 누락 |

## 첫 사용 / 온보딩

### 첫 방문자 여정 (Step-by-step)
1. 사용자가 URL 접속 → 홈 화면 진입
2. 화면 중앙에 URL 입력 폼이 즉시 보임 (`max-w-2xl`)
3. 폼 아래 1줄 안내: "YouTube 영상 URL을 붙여넣으면 댓글을 분석해 리포트를 만들어 드립니다 (약 1분 소요)"
4. 좌측 사이드바는 Empty State ("아직 분석한 영상이 없어요. 위에서 YouTube URL을 붙여넣어 시작하세요.")
5. 사용자가 URL 붙여넣기 → `onPaste` 핸들러가 즉시 형식 검증, 유효하면 "분석 시작" 버튼 활성 + 동일 영상 이력 즉시 검사
6. "분석 시작" 버튼 또는 `Enter` 키 → 진행 단계 표시 시작 (남은 시간 추정 함께)
7. 분석 완료 → 리포트 페이지 자동 이동 + Toast "분석 완료" (2초)

### 재방문자 여정 (Step-by-step)
1. 사용자가 다시 접속 → 사이드바에 히스토리 표시 (최근 분석 우선, 상대시간 라벨)
2. **선택 A**: 사이드바 항목 클릭 → 해당 리포트 페이지로 즉시 라우팅 (스켈레톤 후 순차 fade-in)
3. **선택 B**: 상단 URL 폼에 새 URL 입력 → 첫 방문 흐름과 동일
4. URL이 히스토리에 이미 있는 영상이면 "동일 영상 재분석 안내" 카드 표시 ("기존 리포트 보기" / "새로 분석")

### 별도 도움말/튜토리얼 페이지 없음
도구 자체가 직관적이도록 설계 — 단축키 안내는 폼 placeholder나 hover 툴팁으로 간접 제공.

## 동일 영상 재분석 안내

사용자가 이미 분석한 영상 URL을 다시 입력했을 때:

- 폼 아래 안내 카드:
  - "이 영상은 {N}일 전에 분석했습니다."
  - 버튼 2개: "기존 리포트 보기" / "새로 분석"
- "새로 분석" 선택 시 새 uuid로 별도 저장 (기존 리포트 덮어쓰지 않음)
- `lib/storage.ts`의 `findReportByVideoId(videoId)` 헬퍼로 클라이언트에서 즉시 검사 (Route Handler 호출 없이)

## 사용자 신뢰감

- 리포트 상단 영상 카드에 분석 메타 명시: **"댓글 N개 분석 · {createdAt 상대시간 '몇 분 전' / '어제' / '3일 전'} · Gemini 2.5 Flash"** (ADR-011 — Claude fallback 시 모델명 동적 표시)
- 사이드바 footer 또는 페이지 하단 1줄 안내: **"분석 결과는 이 기기의 브라우저에만 저장됩니다."**
- 분석 진행 중 표시되는 "메타데이터 수집 중 / 댓글 N/200개 수집 중 / 분석 중"도 진행 상황의 투명성을 제공

## 키보드 단축키

| 키 | 동작 | 동작 가능 컨텍스트 |
|---|---|---|
| `Enter` (URL 입력 폼 포커스) | 분석 시작 | 입력 폼 |
| `Esc` | 분석 취소 (`AbortController.abort()`) | 분석 진행 중 |
| `Cmd / Ctrl + S` | 마크다운 다운로드 | `/report/[id]` |
| `Cmd / Ctrl + K` | URL 입력 폼으로 포커스 이동 | 홈 화면 |
| `Cmd / Ctrl + C` (인용 텍스트 선택 시) | 표준 브라우저 복사 | 리포트 페이지 |

플랫폼 분기: `event.metaKey || event.ctrlKey` (macOS Cmd / Windows·Linux Ctrl).

## 붙여넣기 감지 (`onPaste`)

URL 입력 필드에 `onPaste` 핸들러를 등록해 사용자가 URL을 붙여넣는 순간 즉시 반응한다.

- **즉시 형식 검증**: 붙여넣은 텍스트를 `extractVideoId(text)`로 검증
- 유효한 YouTube URL이면:
  - "분석 시작" 버튼 자동 활성 (시각적 강조 — Primary 색상)
  - `findReportByVideoId`로 동일 영상 분석 이력 즉시 검사 → 있으면 "동일 영상 재분석 안내" 카드 표시
- 유효하지 않으면:
  - 입력은 그대로 두되 inline 에러 메시지는 즉시 표시하지 않음 (사용자가 편집 중일 수 있으므로 `blur` 또는 `Enter` 시점에 표시)
- 키보드 단축키 `Cmd/Ctrl + V`로 붙여넣어도 동일하게 동작

## 마이크로 카피 톤 가이드

- **기본 톤**: 간결한 존댓말 ("~하세요", "~합니다", "~할 수 없습니다")
- **명령형 회피**: "지금 분석" → "분석 시작"
- **"AI" / "인공지능" 단어 회피** (UI_GUIDE 안티패턴 — "Powered by AI" 배지 금지와 일관). "분석", "리포트" 사용
- **에러 메시지는 사용자 다음 액션 명시**: "URL을 확인해주세요", "내일 다시 시도해주세요"
- **캐주얼체 회피**: "~했어요" → "~합니다 / ~했습니다"
- **확신 부여**: 결과 도착 시 "분석 완료" (모호한 "끝났어요" 금지)
